## The 4-Part Bidirectional Learning Pattern (Proven)

### Part 1: Student Explores Independently
- **Role**: Active experimenter discovering real problems
- **Deliverable**: Concrete observations (file, notes, comparisons)
- **Duration**: 5-10 minutes
- **Example**: Trace `type()` to discover classes are objects; document findings

### Part 2: AI Teaches
- **Role**: Student learning from AI
- **Structure**: Specific prompt â†’ Expected AI response â†’ Convergence activity
- **Deliverable**: Student's own explanation (2-3 paragraphs)
- **Duration**: 10-15 minutes
- **Example**: Ask AI about metaclass hierarchy; test understanding with follow-up question

### Part 3: Student Challenges AI
- **Role**: Student teaching AI through edge cases
- **Structure**: Three scenarios requiring prediction â†’ Verification by running code
- **Deliverable**: Edge case documentation with predictions and outcomes
- **Duration**: 15-20 minutes
- **Example**: Predict metaclass timing behavior; verify with actual execution

### Part 4: Synthesize into Reference Guide
- **Role**: Knowledge synthesizer creating production patterns
- **Deliverable**: Markdown reference guide file (~1-2 pages)
- **Structure**: 4-6 patterns + best practices + gotchas + AI validation
- **Duration**: 20-30 minutes
- **Example**: Create `metaclass_reference_guide.md` with patterns, decision trees, examples
